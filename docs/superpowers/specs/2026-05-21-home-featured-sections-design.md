# 首页推荐区块同构化设计

**日期：** 2026-05-21

**目标：** 将官网首页的“推荐图片”“推荐歌手”视觉结构统一到“热门电视剧”同一设计语言下，并按已确认约束调整布局：图片区与热门电视剧方向相反，歌手区仅保留竖版海报卡。

## 范围

- 仅覆盖 `src/app/(site)/page.tsx` 首页中的三块推荐区。
- 仅覆盖首页对应的区块结构、卡片结构和相关样式。
- 不修改图片详情页、电视剧详情页、歌手详情页的数据结构和路由。
- 不新增兼容分支，直接按新首页方案落地。

## 已确认决策

1. 采用“整段完全同构”方案。
2. “推荐图片”整段需要与“热门电视剧”共享同一视觉家族，但布局方向相反：
   - 左侧为竖版海报卡矩阵。
   - 右侧为主推荐卡。
3. “推荐歌手”不保留主推荐卡，只保留竖版海报卡矩阵。
4. “推荐图片”“推荐歌手”的卡片样式都要与“热门电视剧”的竖版海报卡完全一致，包括比例、边框、阴影、悬浮上移和底部信息浮层。

## 结构方案

### 1. 热门电视剧

- 保持现有主结构不变，继续作为首页视觉基准。
- 左侧主推荐卡承载：
  - 标题
  - 简介
  - CTA 按钮
- 右侧和覆盖位继续使用现有竖版海报卡样式。

### 2. 推荐图片

- 整段外层切换到与“热门电视剧”一致的 `heroSurface` 家族容器。
- 内层布局改成与“热门电视剧”镜像：
  - 桌面端左侧显示 4 张竖版图片海报卡。
  - 右侧显示 1 张图片主推荐卡。
- 图片主推荐卡负责承载：
  - 标题“推荐图片”
  - 辅助文案“每日精选，灵感无限”
  - CTA 按钮“更多”
- 左侧 4 张图片卡不再使用当前白底信息块方案，统一改成竖版海报卡：
  - 封面铺满
  - hover 时在底部出现半透明信息面板
  - 信息内容为图片标题、标签或描述摘要

### 3. 推荐歌手

- 整段外层也切换到与“热门电视剧”一致的统一推荐容器视觉。
- 不再保留独立主卡和蓝紫色面板。
- 仅渲染竖版歌手海报卡矩阵：
  - 桌面端优先与首页其他推荐区保持统一列距和卡片比例
  - 移动端自动退化为普通双列或单列网格
- 每张歌手卡复用竖版海报卡交互：
  - 封面铺满
  - hover 显示歌手名和引导文案
  - 链接目标保持现有歌手详情页

## 组件与代码组织

1. 在 `src/app/(site)/page.tsx` 中收敛首页推荐卡实现，减少“电视剧卡 / 图片卡 / 歌手卡”三套完全独立结构。
2. 提取共享的竖版海报卡渲染模式，允许仅通过传入链接、背景图和浮层文案完成差异化渲染。
3. 主推荐卡保持按区块定制文案，但共享相同的基础布局骨架和视觉样式。
4. `src/components/site/site-visuals.module.scss` 中优先复用现有：
   - `heroSurface`
   - `promoCard`
   - `heroPoster`
5. 删除或降级仅服务旧版推荐图片/歌手卡的样式依赖，避免首页继续维护两套卡片语言。

## 数据使用

- 继续使用现有首页数据来源：
  - `listFeaturedShows`
  - `listFeaturedImages`
  - `listFeaturedSingers`
- 不改接口签名，不改返回类型。
- 仅在首页对已有结果重新切片：
  - 电视剧：继续保留主卡 + 海报矩阵分配方式
  - 图片：新增“主卡展示项”和“海报矩阵项”的切片分配
  - 歌手：仅保留海报矩阵切片

## 响应式规则

- 移动端不强行保持桌面端覆盖式布局。
- 当屏幕宽度不足时：
  - 电视剧区继续退化为统一卡片网格
  - 图片区退化为主卡在上、海报卡在下，或统一顺序网格
  - 歌手区退化为普通卡片网格
- 所有区块保持统一的卡片圆角、间距和 hover 节奏，避免首页不同分区出现明显断层。

## 非目标

- 不修改导航结构。
- 不修改首页以外的列表页视觉。
- 不引入新的数据字段。
- 不新增轮播、自动切换或复杂动画。

## 验证

- 先补首页结构回归测试，再修改实现。
- 测试至少覆盖：
  - 首页继续存在“热门电视剧”海报卡结构。
  - “推荐图片”使用镜像后的主卡 + 竖版海报卡结构。
  - “推荐歌手”仅保留竖版海报卡结构，不再依赖旧歌手卡壳样式。
- 运行相关测试与类型检查。
- 本次不执行 `lint`。
